HTML Multimedia Elements

Multimedia elements allow you to embed images, audio, video, and external content directly into your web pages.

Images

Basic Image
<img src="image.jpg" alt="Description" width="400" height="300">
Output:

Attributes:

  • src: Image source URL
  • alt: Alternative text (required for accessibility)
  • width/height: Dimensions in pixels
Responsive Images

Use srcset and sizes for different screen resolutions:

<img src="small.jpg" 
     srcset="small.jpg 500w, medium.jpg 1000w, large.jpg 1500w"
     sizes="(max-width: 600px) 480px, 800px"
     alt="Responsive image example">

Audio

Audio Player
<audio controls>
  <source src="audio.mp3" type="audio/mpeg">
  <source src="audio.ogg" type="audio/ogg">
  Your browser does not support the audio element.
</audio>
Output:

Attributes:

  • controls: Shows play/pause controls
  • autoplay: Starts playing automatically
  • loop: Repeats audio
  • muted: Starts muted

Map

Embedded Google Map (Satellite View)
<iframe 
  src="https://www.google.com/maps/embed?pb=!1m18!1m12!1m3!1d3497.627570698462!2d78.03995381507173!3d27.175015983005846!2m3!1f0!2f0!3f0!3m2!1i1024!2i768!4f13.1!3m3!1m2!1s0x3974763b624f3d81%3A0xeffb6edcc8c6e72!2sTaj%20Mahal!5e0!3m2!1sen!2sin!4v1692345678901!5m2!1sen!2sin&t=k" 
  width="600" 
  height="450" 
  style="border:0;" 
  allowfullscreen="" 
  loading="lazy" 
  referrerpolicy="no-referrer-when-downgrade"></iframe>
Output:

Attributes:

  • t=k: Switches to satellite view
  • width / height: Sets the size of the map
  • style="border:0;": Removes the frame border
  • allowfullscreen: Lets user view map in fullscreen
  • loading="lazy": Improves performance by lazy loading
  • referrerpolicy: Controls privacy for requests

Video

Video Player
<video width="640" height="360" controls poster="preview.jpg">
  <source src="video.mp4" type="video/mp4">
  <source src="video.webm" type="video/webm">
  Your browser does not support the video tag.
</video>
Output:

Attributes:

  • poster: Preview image before playback
  • autoplay: Starts playing automatically
  • loop: Repeats video
  • muted: Starts muted

Embedding External Content

YouTube Embed
<iframe width="560" height="315" 
        src="https://www.youtube.com/embed/hmuOic-OZ4U" 
        frameborder="0" 
        allowfullscreen>
</iframe>

Most platforms provide embed codes you can copy/paste. Always check their terms of service.

Image Formats

  • JPEG: Best for photos
  • PNG: Transparency support
  • GIF: Simple animations
  • WebP: Modern compressed format
  • SVG: Vector graphics

Audio Formats

  • MP3: Most compatible
  • WAV: Uncompressed
  • OGG: Open format
  • AAC: Apple standard

Video Formats

  • MP4: Most compatible
  • WebM: Open format
  • OGG: Less common

Semantic HTML Elements

Semantic elements clearly describe their meaning to both browser and developer, improving accessibility and SEO.

Key Semantic Elements

Element Purpose Example
<header> Introductory content or navigation Page header, article heading
<nav> Navigation links Main menu, table of contents
<main> Dominant content of the document Primary content area
<article> Self-contained composition Blog post, news story
<section> Thematic grouping of content Chapter, tabbed content
<aside> Content indirectly related to main Sidebars, pull quotes
<footer> Footer for its nearest ancestor Page footer, article footer

Document Structure Example

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Semantic Example</title>
</head>
<body>
    <header>
        <h1>Website Title</h1>
        <nav>
            <ul>
                <li><a href="#">Home</a></li>
                <li><a href="#">About</a></li>
            </ul>
        </nav>
    </header>
    
    <main>
        <article>
            <h2>Article Title</h2>
            <section>
                <h3>Section 1</h3>
                <p>Content...</p>
            </section>
        </article>
        
        <aside>
            <h2>Related Links</h2>
            <ul>
                <li><a href="#">Link 1</a></li>
            </ul>
        </aside>
    </main>
    
    <footer>
        <p>Copyright © 2023</p>
    </footer>
</body>
</html>

Text-Level Semantics

Element Purpose Example
<mark> Highlighted text Search results
<time> Machine-readable date/time Oct 5
<figure> + <figcaption> Self-contained content with caption Images, diagrams, code snippets
<details> + <summary> Disclosure widget Expandable sections

Benefits of Semantic HTML

  • Accessibility: Screen readers can better interpret your content
  • SEO: Search engines better understand your content structure
  • Maintainability: Easier for developers to read and understand
  • Future-proof: More likely to work with future technologies

Practice Exercise

Create a Multimedia Blog Post

Build an HTML page with the following requirements:

  1. Use proper semantic structure (header, main, article, footer)
  2. Include at least:
    • One image with alt text
    • One audio player
    • One video player
  3. Add a figure with figcaption
  4. Include a details disclosure section
  5. Use at least three text-level semantic elements (mark, time, etc.)

Bonus Challenge

Make the page responsive so media elements resize appropriately on smaller screens.
