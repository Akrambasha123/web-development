HTML Attributes

Attributes are key-value pairs in opening tags (e.g., name="value") that customize tag behavior. They provide extra information like IDs, classes, or sources.

Key Concepts

  • Global attributes (e.g., id, class) work on any tag
  • Specific attributes (e.g., href for <a>) are unique to certain tags
  • They improve functionality, accessibility, and styling

Detailed Explanation

Purpose

Attributes modify default behavior (e.g., set image size) or add features (e.g., require input fields).

Syntax

Attributes always use quotes and can have multiple per tag:

<img src="photo.jpg" alt="Description" width="200" height="150">
Common Global Attributes
  • id - Unique identifier for reference
  • class - Groups elements for CSS/JS
  • title - Provides tooltip text on hover
  • style - Inline CSS (use sparingly)
Why Easy?

Think of attributes as "settings" for HTML elements‚Äîlike adjusting the volume on a TV.

Examples

Link Example
<a href="https://example.com" target="_blank" title="Opens in new tab">Visit Example</a>
Output:
Visit Example

Attributes used:

  • href: Specifies the URL
  • target="_blank": Opens in new window
  • title: Shows text on hover
Input Example
<input type="text" id="username" name="user" required placeholder="Enter username">
Output:

Attributes used:

  • type="text": Creates text input
  • id: Links to label
  • name: Identifies form data
  • required: Makes field mandatory
  • placeholder: Shows hint text

HTML Lists

Lists organize items in structured formats. HTML offers three main types: unordered (bullets), ordered (numbers/letters), and description (term-definition pairs).

Types of Lists

1. Unordered Lists (<ul>)

For items with no specific order (e.g., shopping lists). Uses bullets by default.

<ul>
  <li>Apples</li>
  <li>Bananas</li>
  <li>Oranges</li>
</ul>
Output:
  • Apples
  • Bananas
  • Oranges
2. Ordered Lists (<ol>)

For sequential items (e.g., instructions). Uses numbers by default.

Attributes:

  • type="1/A/a/i/I" - Numbering style
  • start="n" - Begin counting from n
  • reversed - Counts backward
<ol type="A" start="3">
  <li>Preheat oven</li>
  <li>Mix ingredients</li>
</ol>
Output:
  1. Preheat oven
  2. Mix ingredients
3. Description Lists (<dl>)

For term-definition pairs (e.g., glossaries). Uses <dt> (term) and <dd> (definition).

<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language</dd>
  <dt>CSS</dt>
  <dd>Cascading Style Sheets</dd>
</dl>
Output:
HTML
HyperText Markup Language
CSS
Cascading Style Sheets

Nested Lists

Lists can be nested inside other lists to create hierarchies.

<ul>
  <li>Fruits
    <ul>
      <li>Apples</li>
      <li>Bananas</li>
    </ul>
  </li>
  <li>Vegetables
    <ul>
      <li>Carrots</li>
      <li>Broccoli</li>
    </ul>
  </li>
</ul>
Output:
  • Fruits
    • Apples
    • Bananas
  • Vegetables
    • Carrots
    • Broccoli

HTML Tables

Tables display data in rows and columns. They should be used for tabular data, not for page layout.

Basic Table Structure

<table border="1">
  <tr>
    <th>Name</th>
    <th>Age</th>
  </tr>
  <tr>
    <td>John</td>
    <td>25</td>
  </tr>
</table>
Output:
Name Age
John 25

Semantic Table Elements

For better accessibility and styling:

  • <thead> - Groups header content
  • <tbody> - Groups body content
  • <tfoot> - Groups footer content
  • <caption> - Adds a table title
<table>
  <caption>Monthly Sales</caption>
  <thead>
    <tr><th>Month</th><th>Sales</th></tr>
  </thead>
  <tbody>
    <tr><td>January</td><td>$1000</td></tr>
  </tbody>
  <tfoot>
    <tr><td>Total</td><td>$1000</td></tr>
  </tfoot>
</table>
Output:
Monthly Sales
Month Sales
January $1000
Total $1000

Table Attributes

  • colspan="n" - Merge n columns
  • rowspan="n" - Merge n rows
  • border - Shows borders (CSS preferred)
<table border="1">
  <tr>
    <th rowspan="2">Name</th>
    <th colspan="2">Contact</th>
  </tr>
  <tr>
    <th>Email</th>
    <th>Phone</th>
  </tr>
</table>
Output:
Name Contact
Email Phone

HTML Forms

Forms collect user input for processing. The <form> element wraps inputs, and the <input> element creates various field types.

Basic Form Structure

<form action="/submit" method="post">
  <label for="name">Name:</label>
  <input type="text" id="name" name="name" required>
  
  <label for="email">Email:</label>
  <input type="email" id="email" name="email" required>
  
  <input type="submit" value="Submit">
</form>
Output:

Form Attributes

  • action="url" - Where to send form data
  • method="get/post" - How to send data
  • autocomplete - Enable/disable browser autofill
  • novalidate - Skip browser validation

Common Form Elements

  • <label> - Describes inputs (use for="id")
  • <input> - Various field types (see below)
  • <select> - Dropdown menu
  • <textarea> - Multi-line text
  • <button> - Clickable button
  • <fieldset> - Groups related inputs
  • <legend> - Title for fieldset

HTML Input Types

HTML5 introduced many input types that provide better user experience and validation. The type attribute defines the input behavior.

All Input Types with Examples

1. text

Single-line text input

<input type="text" name="text">

2. password

Masked characters for passwords

<input type="password" name="pass">

3. email

Validates email format

<input type="email" name="email">

4. number

Numeric input with stepper

<input type="number" min="1" max="10">

5. tel

Phone number input

<input type="tel" name="phone">

6. url

Validates URL format

<input type="url" name="website">

7. date

Date picker

<input type="date" name="bday">

8. datetime-local

Date and time without timezone

<input type="datetime-local" name="event">

9. month

Month and year picker

<input type="month" name="expiry">

10. week

Week number picker

<input type="week" name="vacation">

11. time

Time picker

<input type="time" name="appt">

12. checkbox

Multiple selections

<input type="checkbox" id="html" name="skill" value="html">

13. radio

Single selection from options

<input type="radio" id="male" name="gender" value="male">

14. file

File upload button

<input type="file" name="upload">

15. submit

Form submission button

<input type="submit" value="Submit">

16. reset

Resets form fields

<input type="reset" value="Reset">

17. button

Custom clickable button

<input type="button" value="Click Me">

18. hidden

Invisible data storage

<input type="hidden" name="user_id" value="123">

19. search

Search field (may show clear button)

<input type="search" name="query">

20. range

Slider control

<input type="range" min="0" max="100">

21. color

Color picker

<input type="color" name="bgcolor">

22. image

Image as submit button

<input type="image" src="submit.png" alt="Submit">

Complete Form Example

<form action="/register" method="post">
  <fieldset>
    <legend>Registration Form</legend>
    
    <label for="username">Username:</label>
    <input type="text" id="username" name="username" required>
    
    <label for="email">Email:</label>
    <input type="email" id="email" name="email" required>
    
    <label for="birthday">Birthday:</label>
    <input type="date" id="birthday" name="birthday">
    
    <label>Gender:</label>
    <input type="radio" id="male" name="gender" value="male">
    <label for="male">Male</label>
    <input type="radio" id="female" name="gender" value="female">
    <label for="female">Female</label>
    
    <label for="interests">Interests:</label>
    <select id="interests" name="interests" multiple>
      <option value="sports">Sports</option>
      <option value="music">Music</option>
    </select>
    
    <label for="comments">Comments:</label>
    <textarea id="comments" name="comments"></textarea>
    
    <input type="submit" value="Register">
    <input type="reset" value="Clear">
  </fieldset>
</form>
Output:
Registration Form
