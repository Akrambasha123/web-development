Introduction to CSS

CSS (Cascading Style Sheets) is used to style HTML elements to control their appearance (colors, fonts, layout, etc.). It separates content (HTML) from design (CSS) for cleaner code and easier updates.

Why Use CSS?

  • Enhances visual appeal - Makes websites more attractive and professional
  • Improves user experience - Creates more readable and navigable interfaces
  • Allows consistent styling - Maintain the same look across multiple pages
  • Saves time - Change the appearance of an entire website by editing one file
  • Provides flexibility - Create different layouts for different devices

Types of Styles Application

Theory: CSS styles can be applied in three main ways: inline (directly on elements), internal (within HTML file), and external (separate file). Each method affects maintenance, reusability, and specificity.

Inline Styles

Theory: Highest specificity, but mixes content and style. Use for quick, one-off changes.

<p style="color: red;">Red text.</p>

Internal Styles

Theory: Placed in <style> tag in <head>. Good for single pages, but not reusable across site. 

<style>
      p { color: blue; }
    </style>

External Styles

Theory: In separate .css file, linked via <link>. Best for large sites: reusable, maintainable.

<link rel="stylesheet" href="styles.css">

styles.css: p { color: green; }

How CSS Works

CSS works by selecting HTML elements and applying styles to them. A CSS rule consists of a selector and a declaration block:

Basic CSS Rule Structure
selector {
  property: value;
  property: value;
}
Example:
h1 {
  color: blue;
  font-size: 24px;
}

This rule selects all <h1> elements and makes their text blue with a font size of 24 pixels.

CSS Syntax and Structure

Understanding CSS syntax is essential for writing effective styles. CSS rules follow a specific pattern that browsers interpret to style HTML elements.

CSS Rule Components

Component Description Example
Selector Targets HTML elements to style h1, .class, #id
Property Style attribute to change color, font-size, background
Value Specific style setting blue, 24px, #ffffff
Declaration Property-value pair color: blue;
Declaration Block One or more declarations { color: blue; font-size: 24px; }

CSS Selectors

Selectors target HTML elements to apply styles. Understanding different selector types and their specificity is key to effective CSS.

Basic Selector Types

Element Selector

Targets all instances of an HTML tag

p { color: blue; }

Selects all <p> elements

Class Selector

Targets elements with a specific class

.highlight { background: yellow; }

Selects <div class="highlight">

ID Selector

Targets a unique element with an ID

#header { text-align: center; }

Selects <div id="header">

Universal Selector

Targets all elements

* { margin: 0; padding: 0; }

Selects every element on the page

Attribute Selector

Targets elements with specific attributes

input[type="text"] { border: 1px solid blue; }

Selects <input type="text">

Grouping Selector

Targets multiple selectors

h1, h2, h3 { color: navy; }

Selects all <h1>, <h2>, and <h3> elements

Pseudo-Class Selectors

Target elements in specific states:

/* Unvisited link */
a:link { color: blue; }
/* Visited link */
a:visited { color: purple; }
/* Mouse over link */
a:hover { color: red; }
/* Selected link */
a:active { color: green; }
/* Focused element */
input:focus { background-color: yellow; }
/* First child element */
li:first-child { font-weight: bold; }
/* Last child element */
li:last-child { font-style: italic; }

Pseudo-Element Selectors

Style specific parts of an element:

/* First letter of a paragraph */
p::first-letter { font-size: 2em; }
/* First line of a paragraph */
p::first-line { font-weight: bold; }
/* Before an element */

/* After an element */
h1::after { content: " ‚òÖ"; }
/* Selection highlight */
::selection { background: yellow; }

Specificity Hierarchy

When multiple rules apply to the same element, specificity determines which rule takes precedence:

  1. Inline styles (highest specificity)
  2. ID selectors
  3. Class, attribute, and pseudo-class selectors
  4. Element and pseudo-element selectors
  5. Universal selector (lowest specificity)

Important Tip: When in doubt, use more specific selectors rather than !important to override styles.

Ways to Apply CSS

CSS can be applied to HTML documents in three different ways, each with its own advantages and use cases.

Inline CSS

Applied directly to HTML elements using the style attribute.

Example
<p style="color: blue; font-size: 16px;">
  This text is blue and 16px.
</p>
Output:

This text is blue and 16px.

Pros: Quick for small changes, highest specificity

Cons: Hard to maintain, not reusable, mixes content with presentation

Internal CSS

Defined within a <style> tag in the HTML document's <head> section.

Example
<!DOCTYPE html>
<html>
<head>
  <style>
    p {
      color: blue;
      font-size: 16px;
    }
    .highlight {
      background-color: yellow;
    }
  </style>
</head>
<body>
  <p>This text is blue and 16px.</p>
  <p class="highlight">This text has a yellow background.</p>
</body>
</html>

Pros: Good for single-page styling, keeps CSS in one place

Cons: Not reusable across multiple pages, increases HTML file size

External CSS

Defined in a separate .css file and linked to HTML documents using the <link> tag.

HTML File (index.html)
<!DOCTYPE html>
<html>
<head>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <p>This text is styled from an external CSS file.</p>
  <p class="highlight">This text has a yellow background.</p>
</body>
</html>
CSS File (styles.css)
p {
  color: blue;
  font-size: 16px;
}
.highlight {
  background-color: yellow;
}

Pros: Reusable across multiple pages, easier maintenance, faster page loading

Cons: Requires additional HTTP request

Best Practice: For most projects, use external CSS files to keep your code organized and maintainable.
